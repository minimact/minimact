How to Build Real-Time Search

Traditional search engines use crawlers that visit websites periodically, checking if content has changed. This approach is:

  • Wasteful (95% of crawls find no changes)
  • Slow (takes 3-7 days to index)
  • Expensive (billions in infrastructure)
  • Carbon-intensive (~1% of global electricity)

The Mactic Approach

Instead of crawlers polling, websites push change notifications:

  1. Client-side tracker detects changes (via MutationObserver + hash checking)
  2. Sends event to Mactic API with category + content
  3. AI generates embeddings for semantic search
  4. Content indexed in 10 seconds
  5. Discoverable immediately via category-scoped search
